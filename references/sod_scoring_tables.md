# S/O/D 评分表与 AP 行动优先级矩阵

> 基于 AIAG & VDA FMEA 手册（2019 版）表 D1/D2/D3 与 AP 矩阵整理。本文件是 FMEA 风险分析的唯一权威评分依据，**严禁脱离此表主观评分**。

---

## 一、严重度 S（Severity）评分表

### 表 D1/P1：产品/过程一般评估标准——严重度（S）

**评分原则**：
- S 评级用于衡量失效影响的严重程度
- 不考虑频度和探测度
- S=9-10 仅用于"安全/法规"场景
- DFMEA 与 PFMEA 的 S 评分必须一致（手册 1.4 节要求）

| S | 等级 | 严重度标准 |
|---|---|---|
| **10** | 非常高 | 影响到车辆和/或其他车辆的操作安全，驾驶员、乘客、道路使用者或行人的健康状况 |
| **9** | 非常高 | 不符合法规 |
| **8** | 高 | 在预期使用寿命内，失去正常驾驶所必需的车辆主要功能 |
| **7** | 高 | 在预期使用寿命内，降低正常驾驶所必需的车辆主要功能 |
| **6** | 中 | 失去车辆次要功能 |
| **5** | 中 | 降低车辆次要功能 |
| **4** | 低 | 外观、声音、振动、粗糙度或触感令人感觉非常不舒服 |
| **3** | 低 | 外观、声音、振动、粗糙度或触感令人感觉中度的不舒服 |
| **2** | 低 | 外观、声音、振动、粗糙度或触感令人略微感觉不舒服 |
| **1** | 非常低 | 没有可觉察到的影响 |

### S 评分实务指南

**S=10 场景**（必须满足"危及人身健康"）：
- 制动失灵、转向失控、加速失控
- 安全气囊误爆或不爆
- 动力电池热失控
- 车门行驶中意外开启
- 转向柱断裂

**S=9 场景**（必须满足"不符合法规"）：
- 排放超国标
- 噪音超国标
- 灯具不符合 ECE/GB 法规
- 后视镜视野不达标
- 制动距离超法规

**S=8 场景**（必须满足"失去主要功能"）：
- 发动机无法启动
- 空调完全不制冷
- 车窗无法升降
- 雨刮完全不工作
- 后视镜无法调节

**S=7 场景**（必须满足"降低主要功能"）：
- 发动机功率下降 10% 以上
- 空调制冷不足（出风口温度高于设计值）
- 雨刮速度变慢

**S=4 场景**（外观非常不舒服）：
- 漆面颗粒密集
- 异响明显
- 振动剧烈

**S=1 场景**（无可觉察影响）：
- 内部不可见的微小划痕
- 客户永远看不到的内部瑕疵

⚠️ **严禁**：
- 把"客户投诉"判为 S=10（投诉不是安全/法规问题）
- 把"外观问题"判为 S=8（外观最高 S=4）
- 把"成本损失"判为高 S（FMEA 只评估技术风险，不评估财务风险）

## 二、频度 O（Occurrence）评分表

### 表 D2/P2：产品/过程一般评估标准——频度（O）

**评分原则**：
- O 评级基于预防控制有效性
- 频度评级是 FMEA 范围内的相对级别，可能不反映实际发生情况
- 应参考类似产品的过往使用情况、保修数据、现场经验
- 严禁无依据给出 O=1-2（必须有充分现场数据）

### O 评分关键问题

团队评级时，回答以下问题：

1. 类似组件/子系统/系统的过往使用情况和现场怎样？
2. 该项目是否是借用件或类似于先前水平的项目？
3. 与先前水平的项目相比，变化有多大？
4. 该项目是完全新的吗？
5. 应用是什么或有哪些环境改变？
6. 是否使用了工程分析（如可靠性）来对预期频度等级进行估计？
7. 预防控制是否到位？
8. 在产品开发过程中，其稳健性是否得到验证？

### O 评分表

| O | 等级 | 失效起因的预测 |
|---|---|---|
| **10** | 极高 | 在无操作经验和/或在运行条件不可控制的情况下的任何地方对新技术的首次应用。没有对产品进行验证和/或确认的经验。不存在标准，且尚未确定最佳实践。预防控制不能预测使用现场绩效或不存在预防控制。 |
| **9** | 非常高 | 在公司内首次应用具有技术创新或材料的设计。新应用，或工作周期/运行条件有改变。没有对产品进行验证和/或确认的经验。预防控制不是针对确定特定要求的性能。 |
| **8** | 高 | 在新应用内首次使用具备创新技术的设计产品或材料。新应用，或工作周期/运行条件有改变。没有对产品进行验证和/或确认的经验。极少存在现有标准和最佳实践，不能直接用于该设计产品。预防控制不能可靠地反映使用现场绩效。 |
| **7** | 高 | 根据相似技术和材料的新型设计。新应用，或工作周期/运行条件有改变。没有对产品进行验证和/或确认的经验。标准、最佳实践和设计规则符合基础设计要求，但不适用于创新产品。预防控制提供了有限的性能指标。 |
| **6** | 中 | 应用现有技术和材料，与之前设计相似。类似应用，工作周期或运行条件有改变。之前的测试或使用现场经验。存在标准和设计规则，但不足以确保不会出现失效起因。预防控制提供了预防失效起因的部分能力。 |
| **5** | 中 | 应用成熟技术和材料，与之前设计相比有细节上的变化。类似的应用、工作周期或运行条件。之前的测试或使用现场经验，或为具有与失效相关测试经验的新设计。在本设计中对最佳实践进行再评估，但尚未经过验证。预防控制能够发现与失效起因相关的产品缺陷，并提供部分性能指标。 |
| **4** | 高 | 与短期现场使用暴露几乎相同的设计，用于类似应用，工作周期或运行条件有细微变化。基于之前测试或使用现场经验，之前设计和为新设计而进行的改变符合最佳实践、标准和规范要求。预防控制能够发现与失效起因相关的产品缺陷，很可能反映设计符合性。 |
| **3** | 低 | 对已知设计（相同应用，在工作周期或操作条件方面）和测试或类似运行条件下的现场经验的细微变化或成功完成测试程序的新设计。考虑到之前设计的经验教训，设计预计符合标准和最佳实践。预防控制能够发现与失效起因相关的产品缺陷，并预测与生产设计的一致性。 |
| **2** | 非常低 | 与长期现场暴露几乎相同的设计。相同应用，具备类似的工作周期或运行条件。在类似运行条件下的测试或使用现场经验。考虑到之前设计的经验教训并对其具备充足的信心，设计预计符合标准和最佳实践。预防控制能够发现与失效起因相关的产品缺陷，并显示出对设计符合性的信心。 |
| **1** | 非常低 | 通过预防控制完全消除失效起因。 |

### O 评分实务指南

**O=10 场景**（必须满足"新技术首次应用"）：
- 全新电池化学体系（如固态电池首次量产）
- 全新电机拓扑（如轴向磁通电机首次应用）
- 全新材料（如碳纤维首次用于车身结构件）

**O=9 场景**（公司首次应用创新）：
- 公司首次使用 SiC 功率器件
- 公司首次使用 800V 高压架构
- 公司首次使用激光雷达

**O=5-6 场景**（最常见量产产品）：
- 成熟 IGBT 模组 + 细节优化
- 成熟注塑工艺 + 新产品形状
- 成熟焊接工艺 + 新材料牌号

**O=1-2 场景**（必须有充分数据）：
- 同一产品已量产 5 年以上，无相关失效
- 同一供应商、同一工艺、同一材料，仅换型号标识

⚠️ **严禁**：
- 把"已量产 10 年的成熟产品"判为 O=10
- 把"新技术首次应用"判为 O=1-2（无现场数据支撑）
- 把"我们觉得不会发生"判为低 O（必须有客观依据）

## 三、探测度 D（Detection）评分表

### 表 D3/P3：产品/过程一般评估标准——探测度（D）

**评分原则**：
- D 评级基于探测方法成熟度 + 探测机会
- 探测时机越早（设计阶段 > 试产阶段 > 量产阶段），D 越低
- 探测方法越成熟（已验证 > 新方法 > 无方法），D 越低
- D=1 仅用于"测试证明不会失效"或"探测方法总是能探测到"

### D 评分关键问题

1. 当前探测方法是什么？
2. 探测方法是否经过验证？
3. 探测时间点（在设计阶段/试产阶段/量产阶段/客户使用阶段）？
4. 探测能力（通过/不通过 vs. 失效测试 vs. 老化测试）？
5. 探测覆盖率（100% 全检 vs. 抽检）？

### D 评分表

| D | 探测能力 | 探测方法成熟度 | 探测机会 |
|---|---|---|---|
| **10** | 非常低 | — | 尚未制定测试过程 |
| **9** | 非常低 | 没有为探测失效模式或失效起因而特别地设计测试方法 | 通过/不通过测试、失效测试、老化测试 |
| **8** | 低 | 新测试方法，尚未经过验证 | 通过/不通过测试、失效测试、老化测试 |
| **7** | 低 | 已经验证的测试方法，该方法用于功能性验证或性能、质量、可靠性以及耐久性确认；测试计划时间在产品开发周期内较迟，如果测试失败将导致重新设计、重新开发模具导致生产延迟 | 通过/不通过测试 |
| **6** | 中 | （已验证方法，时间适中） | 失效测试 |
| **5** | 中 | （已验证方法，时间适中） | 老化测试 |
| **4** | 高 | 已经验证的测试方法，该方法用于功能性验证或性能、质量、可靠性以及耐久性确认；计划时间充分，可以在开始生产之前修改生产工装 | 通过/不通过测试 |
| **3** | 高 | （已验证方法，时间充分） | 失效测试 |
| **2** | 非常高 | （已验证方法，时间充分） | 老化测试 |
| **1** | 非常高 | 之前测试证明不会出现失效模式或失效起因，或者探测方法经过实践验证总是能够探测到失效模式或失效起因 | — |

### D 评分实务指南

**D=10 场景**：
- 完全没有任何测试计划
- 失效模式是新发现的，从未考虑过测试

**D=9 场景**：
- 有通用的功能测试，但未针对该特定失效模式设计
- 例：测试了 LED 是否点亮，但没测试高温下的光衰

**D=7-8 场景**：
- 测试方法新开发，未经过验证
- 测试时间太晚（量产阶段才发现，已无法修改设计）

**D=5-6 场景**：
- 测试方法已验证，但只是通过/不通过测试
- 测试时间适中（试产阶段）

**D=2-4 场景**：
- 测试方法已验证
- 测试时间充分（设计阶段或试产早期）
- 失效测试或老化测试（比通过/不通过更敏感）

**D=1 场景**：
- 测试方法经过长期实践验证
- 100% 探测率（如 100% 在线自动检测）

⚠️ **严禁**：
- 把"100% 在线自动检测"判为 D=10（逻辑矛盾）
- 把"无任何测试"判为 D=1（逻辑矛盾）
- 把"通过/不通过测试"判为 D=1-2（通过/不通过最多 D=4）

## 四、AP 行动优先级（Action Priority）矩阵

### 4.1 AP 三档定义

| AP | 处置原则 |
|---|---|
| **H（高）** | 评审和措施的最高优先级。团队需要确定适当的措施来改进预防和/或探测控制，或证明并记录为何当前的控制足够有效。 |
| **M（中）** | 评审和措施的中等优先级。团队应该确定适当的措施来改进预防和/或探测控制，或由公司自行决定，证明并记录当前的控制足够有效。 |
| **L（低）** | 评审和措施的低优先级。团队可以确定措施来改进预防或探测控制。 |

### 4.2 AP 判定逻辑

**优先级顺序**：S（严重度）→ O（频度）→ D（探测度）

1. **第一优先：S 越高，AP 越倾向于 H**
   - S=9-10：几乎必为 H（除非 O=1 且 D=1 才为 M）
   - S=6-8：AP 取决于 O 和 D
   - S=2-3：多为 L，少数情况为 M
   - S=1：必为 L

2. **第二优先：O 越高，AP 越倾向于 H**
   - S 相同时，O 高者 AP 更高

3. **第三优先：D 越高，AP 越倾向于 H**
   - S、O 相同时，D 高者 AP 更高

### 4.4 特殊规则

1. **S=9-10 强制管理层评审**：
   - 对于 S=9-10 且 AP=H/M 的失效影响，必须由管理层评审
   - 包括所采取的任何建议措施

2. **AP 与 RPN 的关系**：
   - 2019 版明确**不推荐使用 RPN 阈值**
   - AP 表与 RPN 不是简单映射关系
   - 同一 AP 等级可能对应不同的 RPN

3. **DFMEA 与 PFMEA 的 AP 表相同**：
   - 本表适用于 DFMEA 和 PFMEA
   - FMEA-MSR 的 AP 表不同（本 Skill 暂不覆盖）

4. **AP 表不可修改**：
   - 如果公司修改 S/O/D 表，AP 表也需相应评审
   - 但本 Skill 严格使用 2019 版原表，不允许修改

### 4.5 AP 评分实务示例

**示例 1**：S=10, O=4, D=6
- 判定：**H**
- 依据：S=10 几乎必为 H（除非 O=1 且 D=1）

**示例 2**：S=8, O=2, D=3
- 判定：**M**
- 依据：S=8 且 O=2-3 → M

**示例 3**：S=5, O=8, D=8
- 判定：**H**
- 依据：S=5 + O=6-10 + D=7-10 → H

**示例 4**：S=2, O=10, D=10
- 判定：**M**
- 依据：S=2 + O=6-10 + D=7-10 → M

**示例 5**：S=1, O=10, D=10
- 判定：**L**
- 依据：S=1 必为 L

## 五、RPN vs AP 对比

### RPN（旧版，已废弃）

- 公式：RPN = S × O × D
- 范围：1-1000
- 缺陷：
  - 对 S/O/D 权重相等，无法体现 S 优先
  - 不同 S/O/D 组合可能产生相同 RPN
  - 团队无法确定优先级

### AP（2019 版）

- 三档：H / M / L
- 优先级：S > O > D
- 优势：
  - 明确 S 优先（安全/法规最重要）
  - 1000 种组合有明确判定
  - 团队易于决策

### 转换关系

| RPN 范围 | 可能的 AP |
|---|---|
| 1-50 | L / M（多数 L） |
| 50-200 | L / M / H（混合） |
| 200-500 | M / H（多数 M 或 H） |
| 500-1000 | H（多数 H） |

⚠️ **本 Skill 严禁输出 RPN 评分**，所有风险评级必须使用 AP。

## 六、评分一致性检查

### 6.1 DFMEA 与 PFMEA 的 S 一致性

- 同一失效影响在 DFMEA 和 PFMEA 中的 S 评分必须相同
- 例：制动失灵 → DFMEA 的 S=10，PFMEA 的 S 也必须=10

### 6.2 同一失效链的 S/O/D 逻辑一致性

- S=10（影响安全） + O=1（不会发生） + D=10（无法探测） → 逻辑矛盾
- S=10 + O=10 + D=10 + AP=L → 逻辑矛盾（S=10 必为 H 或 M）

### 6.3 预防控制与 O 评分一致性

- PC 充分 → O 应较低（1-4）
- PC 不足 → O 应较高（5-10）
- 严禁 PC 充分但 O=10

### 6.4 探测控制与 D 评分一致性

- DC 充分 → D 应较低（1-4）
- DC 不足 → D 应较高（5-10）
- 严禁 DC 充分但 D=10
